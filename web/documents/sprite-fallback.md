# ระบบ Sprite Fallback (Sprite Fallback System)

เอกสารนี้อธิบายกลไกการค้นหาและทดแทนรูปภาพ (Sprite Fallback) ของสัตว์เลี้ยงในเกม Tamagotchi ซึ่งถูกจัดการโดยไฟล์ `helpers/petAssets.js` กลไกนี้ถูกออกแบบมาแบบลดหลั่นกันเป็นชั้นๆ (Cascade) เพื่อให้ระบบสามารถแสดงผลรูปภาพสัตว์เลี้ยงได้เสมอ แม้ว่าไฟล์ภาพหรือการตั้งค่าสำหรับสายพันธุ์หรือช่วงวัยนั้นๆ จะยังไม่สมบูรณ์ก็ตาม

ระบบ Fallback แบ่งออกเป็น 3 ระดับหลัก ดังนี้:

## 1. Fallback ระดับสายพันธุ์ (Pet ID Level)
เมื่อเกมต้องการแสดงผลสัตว์เลี้ยง ระบบจะตรวจสอบก่อนว่า `petId` นั้นมีการตั้งค่าไว้ใน `PET_CATALOG` หรือไม่
- หากมีการลงทะเบียนไว้ ระบบจะดำเนินงานต่อตามปกติ
- หาก **ไม่พบ** สายพันธุ์นั้น ระบบจะวิ่งกลับไปใช้ค่า `DEFAULT_PET_ID` (`classic`) โดยอัตโนมัติ

*ตัวอย่าง: หากเรียกใช้ `petId: "dragon"` ซึ่งไม่มีอยู่จริง ระบบจะใช้สายพันธุ์ `"classic"` แทนทั้งหมด*

## 2. Fallback ระดับช่วงวัย (Stage Level)
เมื่อผ่านระดับสายพันธุ์มาแล้ว ระบบจะตรวจสอบช่วงวัย (Stage) ที่ต้องการ เช่น `egg`, `baby`, `child`, `teen`, `adult` 

**สำหรับสายพันธุ์ที่เป็น Single Stage (สายพันธุ์ใหม่ทั่วไป):**
- ระบบจะมองข้ามเรื่องช่วงวัยไปเลย และยึดการใช้รูปภาพจากสายพันธุ์นั้นเป็นหลัก ไม่ว่าจะอยู่ในช่วงอายุใดก็ตาม

**สำหรับสายพันธุ์ที่มีหลายช่วงวัย (Multi-stage อย่างเช่น `classic`):**
หากสายพันธุ์นั้นๆ ไม่มีการตั้งค่าช่วงวัยดังกล่าว (อ้างอิงจากฟังก์ชัน `getStageCatalog`) รูปแบบการ Fallback จะทำงานตามลำดับดังนี้:
1. ค้นหาการตั้งค่าของวัยที่ระบุในสายพันธุ์นั้นๆ
2. หากไม่มี จะค้นหาการตั้งค่าของวัยตั้งต้น `FALLBACK_STAGE` (`child`) ในสายพันธุ์นั้นๆ
3. หากยังไม่มีอีก จะไปดึงการตั้งค่าของวัย `FALLBACK_STAGE` (`child`) ของสายพันธุ์ `classic` มาใช้แทน

*ตัวอย่าง: หากเรียกใช้สายพันธุ์ใหม่ที่เป็น Single Stage แม้อายุในเกมจะเป็น `teen` แต่อาร์ตเวิร์กก็จะถูกดึงจากสายพันธุ์นั้นโดยตรงไม่ต้องนำไปเทียบกับ Stage อีก*

## 3. Fallback ระดับท่าทาง (Variant/Animation Level)
นอกจากเรื่องช่วงวัยแล้ว แต่ละวัยยังมี "ท่าทาง" ย่อยๆ (Variants) เช่น `idle`, `angry`, `sick`, `attack`, `dead`
เมื่อต้องการโหลดภาพ (อ้างอิงจากฟังก์ชัน `getTextureCandidateDescriptors`) ระบบจะลิสต์ลำดับความสำคัญของไฟล์ที่จะนำมาแสดงผล หากหาภาพแรกสุดไม่เจอ มันจะไล่ใช้ภาพในลำดับถัดไปเรื่อยๆ จนกว่าจะเจอ:

1. **Target Variant:** หารูปท่าทางเป้าหมาย (เช่น `sick`) ของตัวมันเอง
2. **Local Idle:** หากไม่พบ จะหารูป `idle` (ท่ายืนปกติ) ของตัวมันเองมาใช้แทน
3. **Classic Target Variant:** หากยังไม่มีอีก (เช่นไม่มีรูปเลย) จะไปหารูปท่าทางเป้าหมาย (เช่น `sick`) ของตัว `classic` ในช่วงอายุเดียวกัน
4. **Classic Idle:** หากข้อ 3 ไม่มี ให้ใช้ `idle` ของตัว `classic` ในช่วงอายุเดียวกัน
5. **Classic Child Target Variant:** หากยังไม่มีอีก ให้ใช้รูปท่าทางเป้าหมาย ของตัว `classic` ในวัย `child` (Fallback stage)
6. **Classic Child Idle:** ตัวเลือกสุดท้าย (ไม้ตายสุดยอด) คือใช้รูป `idle` ของตัว `classic` ในวัย `child`

## กรณีศึกษา (Case Study)
สมมติว่ามีสายพันธุ์ใหม่ชื่อ **`specie1`** ซึ่งเป็นแบบ Single Stage (วัยเดียว) และมีภาพเพียงแค่ท่า `idle.svg` เท่านั้น
- **เมื่อ `specie1` อายุใดๆ ก็ตาม (เช่น `teen`) และเกิดอาการหิว (`angry`):**
  - ระบบจะหา `specie1` > `angry.svg` -> (ไม่พบภาพ)
  - ระบบตกมาที่ลำดับสำรองที่ 1 คือหา `specie1` > `idle.svg` -> (พบภาพ!)
  - **ผลลัพธ์:** สัตว์เลี้ยงจะแสดงภาพยืนปกติแม้ว่าจะอยู่ในสภาวะโกรธก็ตาม
- **เมื่อเราเรียกใช้สายพันธุ์ที่ไม่เคยประกาศภาพใดๆ เลย (เช่น `specie3`):**
  - ตัวเกมจะไม่พบภาพใดๆ เลยในสายพันธุ์นั้น
  - ระบบ Fallback จะหลุดจากระดับตัวมันเอง ไปหาระดับค่าที่ตั้งไว้ (Classic) โดยระบบจะเช็คช่วงอายุของเกมตอนนั้นด้วย 
  - หากตอนนั้นอายุ `baby` ตัวเกมจะไปดึงโครงสร้างของ `classic` มาเทียบ และดึงภาพวัย `child` ของ Classic มาใช้งาน
  - **ผลลัพธ์:** หน้าตาจะกลับไปเหมือนสัตว์เลี้ยงคลาสสิกตัวดั้งเดิมทุกประการ ในวัยตามอายุนั้นๆ
